# 다대다 면접 시뮬레이션 시스템 아키텍처

## 🎯 **프로젝트 핵심 목적**
**다대다 면접 시뮬레이션이 프로젝트의 최종 핵심 목적** - 사용자가 여러 AI 지원자들과 함께 경쟁하는 면접 시스템

## 📊 **현재 상태**
- ✅ **현재**: 일대일 면접만 구현되어 있음
- ⚠️ **목표**: 다대다 기능은 미구현 상태로 구현 필요

## 🏗️ **확정된 기술 스택**
- **Frontend**: React (반응형 다대다 면접 UI)
- **Backend**: FastAPI (비동기 처리, WebSocket 지원)
- **Database**: Supabase (실시간 데이터 동기화)
- **Container**: Docker (컨테이너화)
- **Deploy**: AWS (클라우드 배포)
- **Storage**: AWS S3 (대용량 파일 - 영상, 음성, 문서)

```
AI 지원자 생성 ← External AI Services ← AI 지원자 모델 ← 질문 처리
                                         ↓
답변 평가 → CompetitiveEvaluator → 실시간 순위 계산 → Redis Cache
                                         ↓
결과 저장 → Supabase DB → 피드백 생성 → WebSocket → React UI
```

```
        │
        ├─── QuestionQueueManager ──── 질문 순서 및 난이도 관리
        │
        ├─── TurnOrderManager ──── 실시간 턴 순서 제어
        │
        └─── CompetitiveEvaluator ──── 답변 비교 및 순위 계산
```
